


# 事务


	事务需要满足ACID四个特性。


# ACID


A

	A(atomicity) 
	原子性
	事务里面的操作，要么全部成功执行，要么全部失败回滚，不可以只执行其中的一部分。

C

	C(consistency) 
	一致性
	一个事务的执行不应该破坏数据库的完整性约束。

I	
	
	I(isolation) 
	隔离性
	事务之间的行为不应该互相影响。

D

	D(durability) 
	持久性
	事务一旦提交，结果就是永久的.
	需要将提交的事务持久化到磁盘。
	即使系统崩溃，提交的数据也不应该丢失。
